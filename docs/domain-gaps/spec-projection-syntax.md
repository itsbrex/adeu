# Adeu Projection Dialect (Syntax Specification)

This document defines the "Adeu Projection Dialect" — a superset of GitHub Flavored Markdown used to project complex OOXML structures into a flat, LLM-editable format. 

This specification dictates exactly what the LLM sees in the `read_docx` output and what syntactic rules it must follow when using `process_document_batch`.

## 1. Editable Inline Elements

### 1.1 Footnotes and Endnotes
*   **Syntax (Inline):** `[^fn-{id}]` for footnotes, `[^en-{id}]` for endnotes.
    *   *Note:* The `{id}` uses the stable OOXML `w:id` (e.g., `[^fn-3]`), **not** the rendered display number (e.g., `[^1]`). This guarantees that inserting a new footnote does not invalidate the ID of subsequent footnotes during an editing session.
*   **Syntax (Bottom of document):**
    ```markdown
    ---
    ## Footnotes
    [^fn-3]: The actual footnote text.
    ```
*   **Editability:** Fully bi-directional. Using `ModifyText` on the footnote content at the bottom of the document will apply tracked changes natively inside the `footnotes.xml` / `endnotes.xml` parts.

### 1.2 External Hyperlinks
*   **Syntax:** Standard Markdown links: `[Visible Text](https://url.com)`.
*   **Editability:** 
    *   Editing the `Visible Text` applies standard tracked changes (redlines) to the display text.
    *   Editing the `(https://url.com)` URL updates the link target silently in the DOCX relationship files (no redlines emitted, but surfaced in the silent-change diff).

## 2. Read-Only Inline Elements (Computed Fields)

### 2.1 Cross-References
*   **Syntax:** `[~Visible Text~](#_Ref12345)`
*   **Definition:** The `[~...~]` wrapper indicates that the text is **Computed / Read-Only**. The display text is generated by Word (e.g., "Section 5") and might be natively stale if the document was edited without refreshing fields.
*   **Editability:** STRICTLY READ-ONLY.
    *   Attempting to use `ModifyText` to change the `~Visible Text~` will be rejected by the engine.
    *   Attempting to use `ModifyText` to change the `#_Ref12345` target hash will be rejected.
    *   *Rule for LLMs:* To change what a cross-reference says, edit the target heading or paragraph it points to.

### 2.2 Table of Contents (TOC) / Table of Authorities
*   **Syntax:** `[~Table of Contents — 47 entries~]`
*   **Definition:** Replaces the entire multi-page TOC field block in the body text with a single placeholder.
*   **Editability:** STRICTLY READ-ONLY. Do not attempt to edit the TOC entries. Update the source headings instead.

## 3. The Structural Appendix

To prevent LLMs from accidentally corrupting the document's reference graph, structural metadata is never injected into the editable body text. Instead, it is appended to the absolute bottom of the document.

### 3.1 Boundary Marker
The Appendix is always preceded by this exact system boundary:
```markdown


---

<!-- READONLY_BOUNDARY_START -->
# Document Structure (Read-Only)
The content below is metadata describing the document's reference structure. Do not include this section in any tracked changes or edits — it is for your context only and will be discarded on write.
```
*   **Engine Rule:** Any `ModifyText` targeting text *after* this boundary marker must be immediately rejected by the batch validator.

### 3.2 Defined Terms (Domain Concept)
Heuristically detected terms that carry specific contractual meaning.
```markdown
## Defined Terms
- "Affiliate" — defined in Section 2, used 47 times.
- "Confidential Information" — defined in Section 2, used 23 times.
```

### 3.3 Named Anchors & Back-References (OOXML Concept)
The exact bookmark IDs used for cross-referencing, along with a dependency graph of which clauses point to them.
```markdown
## Named Anchors
- _Ref12345 → Anchored to: "Section 5. Indemnification"
  - Referenced from: Section 7.2, Section 9.4, Schedule B
- _Ref67890 → Anchored to: "Section 7. Termination"
  - Referenced from: Section 9.1
```
